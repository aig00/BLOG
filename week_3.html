OJT Week 03 — In Progress
01

Week Overview

Week 3 was heavily focused on the actual implementation of the Asset Management System. With the requirements gathered and the tech stack selected, it was time to dive into writing code.

I kicked off the week by setting up the foundational architecture, specifically initializing a React.js project and configuring the PostgreSQL database via Supabase. The bulk of the week was then dedicated to building the frontend user interface using React and Tailwind CSS to ensure a clean and responsive design.

// key insight

"Building a user interface isn't just about making it look good; it's about continuously validating the design with end-users to ensure it actually solves their workflow problems."

By the end of the week, I presented the progress to my mentor and the Accounting Department to gather early feedback, ensuring the development stays aligned with their expectations.

05

Reflections & Next Steps

Week 3 was immensely fulfilling. Moving from the theoretical planning phase to writing functional code always feels like a leap forward. Watching the application's interface come alive through React and Tailwind CSS provided a significant confidence boost in my development abilities.

The consultation session with the Accounting Department on Friday was pivotal. It validated some of my design choices while highlighting user workflow preferences I hadn't considered. This proved to me that frequent stakeholder communication is just as critical as the code itself.
